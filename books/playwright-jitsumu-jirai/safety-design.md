---
title: "第4章 4 大防御の体系 — 未知のサイトへの当たり所"
---

第 3 章の 25 件は、抽象化すると 4 つの「防御カテゴリ」に収まります。未知のサイトで詰まったとき、症状からどのカテゴリかを当てれば、対処の方向が決まります。

## 防御1：イベントの信頼性（trusted）

**サイトが見ているもの**：`event.isTrusted`

**詰まる症状**：click が無視される / menu が開かない / 状態が変わらない

**当たり所**：
- 状態変更系は `page.evaluate` の `.click()` を捨て、`locator.click()` に統一
- 「効いていないのに成功する」ので、必ず第 2 章の状態 verify とセットで使う

**判定フロー**：
```
click したのに変化なし
  └ locator.click() に変えて直る？ → trusted 問題で確定
      └ 直らない → 防御2（検出）か防御4（状態分離）を疑う
```

## 防御2：自動化の検出（detection）

**サイトが見ているもの**：UserAgent、`navigator.webdriver`、headless 特有の指紋

**詰まる症状**：ログインは通るのに画面が描画されない / 403 / challenge

**当たり所**：
- まず UA を通常 Chrome に偽装（最頻・最安の対処）
- それでも 403 が続くなら Cloudflare 等の challenge → 実ブラウザ + 通過待ち
- `navigator.webdriver` を気にするサイトには `--disable-blink-features=AutomationControlled` 等のフラグ

**重要な線引き**：これは「自分のアカウントで自分の作業を自動化する」範囲の話です。UA 偽装でアクセス制限や利用規約を回避してまでやるべきではありません。詰まったら「そもそもこのサイトは自動操作を許容しているか（API はないか、規約はどうか）」を先に確認してください。

## 防御3：レンダリングのタイミング（render timing）

**サイトが見ているもの**：特になし（純粋に非同期描画の問題）

**詰まる症状**：要素が見つからない / 一部しか取れない / loading のまま

**当たり所**：
- `waitForTimeout` の固定待ちではなく `waitForSelector` で「出るまで待つ」
- 一覧は lazy render を疑い、「見えている分」前提のコードを書かない（周回方式）
- modal は出現も消滅も待つ。背景レイヤーの残留を assert

## 防御4：見た目と状態の分離（view vs state）

**サイトが見ているもの**：自身の内部状態（フレームワークの state、サーバーの status）

**詰まる症状**：DOM は正しいのに動作が伴わない / カウンタが合わない / 下書きが復元されない

**当たり所**：
- DOM を数えず、内部 API の status を数える
- custom component は実体（input）でなく操作面（label）を叩く
- 「保存した」を信用せず、再取得して確認

## 4 カテゴリ早見表

| 症状 | 第一容疑 | 最初に試すこと |
|---|---|---|
| click が無視される | 防御1 | locator.click() |
| ログイン済なのに描画されない | 防御2 | UA 偽装 |
| 403 / challenge | 防御2 | 実ブラウザ + 待ち |
| 要素が見つからない | 防御3 | waitForSelector |
| 一覧が途中までしか取れない | 防御3 | 周回方式 |
| 選択カウンタが合わない | 防御4 | label click |
| 成功なのに変化なし | 防御1+4 | 状態 verify |

## 設計指針：4 防御を「事前に」織り込む

新しいサイトの自動化を始めるとき、私は最初から次を入れます。

1. context に UA 偽装（防御2の先回り）
2. 主要要素は `waitForSelector`（防御3の先回り）
3. 状態変更系は最初から `locator.click()`（防御1の先回り）
4. 操作後に「状態を数える関数」を 1 つ用意（防御4の常設）

この 4 点を最初に入れておくと、25 件の地雷の大半は「踏む前に避けられる」状態になります。

次章は、この上に乗せる「不可逆操作の安全設計」です。削除のような取り返しのつかない操作を、人間が見ていない時間に任せるための作法を扱います。
